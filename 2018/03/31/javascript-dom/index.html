Javascript | 문서 객체 모델 (DOM)

2018-03-31 13:04:59
Javascript>Basic

간단히 말해 문서 객체 모델(DOM)은 문서내에 있는 내용을 뽑아내서 개념화하는 방법이다.
현실 세계를 예로 들어보면 문서 객체 모델을 세상 객체 모델이라 불러보자.
현실 세계에서 객체란 “차”, “집”, “나무”와 같은 식별 가능한 물건을 말한다.

세상 객체 모델에는 꼭 실체가 있는 것뿐만 아니라 개념도 설명할 수 있다.

예를들어 “왼쪽에 있는 세 번째 집”이라고 하자.
이 설명을 정확히 이해하려면 “왼쪽”과 “세 번째”가 명확해야 한다.
만약 수를 셀 수 없는 사람이거나 반대 방향에 서 있었다면 이해한 의미는 전달 하는 봐와 다른 것이 된다.

웹 페이지에서도 비슷한 상황이 나타난다.
초기 자바스크립트 버전은 개발자들이 그림이나 폼 양식 같은 구체적인 문서 내용을 다루게 하였다.
그림이나 폼 양식은 웹 페이지에 이미 정의되어 있어서 사용하기에 편리했기 때문이다.
자바스크립트에서는 “문서에서 세 번째 그림”이라든가 “detail라고 부르는 폼 양식”을 다음과 같이 표시할 수 있었다.

이런 임시적인 형식을 가진 초기 문서 구조를 DOM 레빌 0라고 불렀다.

이 DOM이 점차 발전하여 현재의 모든 브라우저에서 사용하는 표준이 되었다.

문서 (Document)

문서 객체 모델(DOM)은 문서(document)가 없으면 움직이지 않는다.
웹 문서를 만들어 웹 브라우저에 띄우는 순간 작성한 문서가 객체로 바뀌게 되어 DOM이 살아 움직이기 시작한다.

객체(Object)

자바스크립트에는 세 종류의 객체가 있다.

  • 프로그래머가 만든 사용자 정의 객체
  • 배열, 수학, 날짜와 같이 자바스크립트에서 이미 만들어진 객체
  • 웹 브라우저가 제공하는 주요 객체

자바스크립트 초기에는 중요한 몇 가지 주요 객체들이 스크립트 제작에 사용됐다.
그 중 가장 기본적인 것이 바로 윈도우 객체(window object)이다.
이러한 객체는 웹 브라우저 창 내부의 속성을 표현하는 것이다.
따라서 윈도우 객체의 프로퍼티와 메서드는 브라우저 객체 모델(BOM, Browser Object Model)이라고 부르기도 한다. (의미적으론 윈도우 객체 모델)

모델(Model)

DOM에서 M은 모형(Model)을 말한다.
모형이라는 단어보다는 지도라는 단어가 더 적합하다.
지도처럼 모형은 특정 사물을 대신 나타내는 것이다.
문서 객체 모델은 웹 브라우저 창 안에서 읽은 웹 문서를 나타내 주는 것이다.
브라우저는 이 문서의 ‘지도’를 제공해주고 이 지도를 자바스크립트를 통해 읽거나 사용할 수 있다.
DOM에서 사용하는 가장 중요한 규칙은 문서를 나뭇가지(tree)형 구조로 표시한다는 것이다.
(가계도 같은 구조)

이것은 아래와 같은 그림으로 표현이 가능하다.

doctype을 선언해주고 난 후 문서는 <html>을 여는 것으로 시작된다.
웹 페이지 내의 모든 요소들은 <html> 안에 있다.
그러므로 <html> 모든 요소들의 뿌리 또는 모든 요소의 ‘부모’라 할 수 있다.

<html> 은 뿌리 요소이고 문서의 시작이다.
한 단계 내려가면 <head>와 <body>라는 두 개의 ‘가지’를 볼 수 있다.
가지가 나란하니 두 요소는 ‘형제’라 볼 수 있다.
그리고 모두 ‘자식’을 거느리고 있으므로 ‘부모’가 될 수 있다.
<head>는 <meta>와 <title>을 가지고 있고 ‘부모’이다.

이러한 가계도처럼 나타나는 것을 노드 트리라고 한다.

노드 (node)

노드(node)라는 용어는 어떤 연결망에서 특정 지점과 지점을 연결하는 데 표시하는 것을 말한다.
즉 네트워크는 노드의 집합을 의미한다.

현실 세계의 모든 것은 원자로 구성되어 있다.
원자는 세상의 노드라고 할 수 있다.
원자는 더 작게 쪼개면 더 작은 양성자나 전자 같은 것으로 구성되는데 이것도 노드라고 부를 수 있다.

DOM에서도 문서는 노드의 집합이다.
문서라는 ‘나무’ 위에 ‘가지’나 잎 같은 노드를 갖고 있는 것이다.

- 요소 노드 (element)

DOM에서 원자와 같은 것이 요소(element) 노드이다.
위에서 쇼핑목록 문서의 구조를 볼 때 <body>, <p>. <ul> 같은 요소로 표현하였다.

요소는 웹 상에서 제공되는 문서의 가장 기본적인 벽돌 조각을 의미한다.
문서의 구조라 하면 이런 요소가 배열된 것을 뜻한다.

태그는 요소의 이름을 붙인 것이다.
단락 요소에는 p라는 이름을,
순서 없는 목록을 ul이라는 이름을 가지고 있다.

요소는 다른 요소를 포함할 수 있다.
모든 <li>요소는 <ul>요소 안에 포함되어 있다.

- 텍스트 노드 (text)

요소는 여러가지 노드 형식 중 하나이다.
문서가 순수하게 요소로만 구성되어 있다면 구조는 가질지 모르지만 내용은 없다.
웹 문서에는 내용이 들어 있어야한다.
대부분의 내용은 텍스트로 제공할 수 있다.

예를 들어 <p>요소에는 “이 물건 사는 것을 잊지 마세요.”라는 내용이 들어있다.
이것이 바로 텍스트(text) 노드이다.

(X)HTML에서 텍스트 노드는 항상 요소 노드 안에 포함된다.
그렇지만 모든 요소가 텍스트 노드를 포함하는 것은 아니다.
<ul>요소는 텍스트를 직접적으로 포함하고 있지 않다.
단지 텍스트 노드를 포함하고 있는 <li>요소를 포함하고 있을 뿐이다.

- 속성 노드 (attribute)

속성이라는 것은 요소에 대한 정확한 정보를 표현할 때 사용한다.

예를 들어 title 속성을 보면 요소가 포함하고 있는 것을 좀더 정확히 표현하는 수단으로 사용한다.


문서 객체 모델에서 title=”a gentle reminder”는 속성(attribute) 노드이다.
속성들은 항상 태그를 열 때 쓰여지기 때문에 항상 요소 노드 안에 포함된다.

모든 요소는 속성을 포함하지는 않지만,
모든 속성은 요소안에 포함되어야 한다.

- CSS와의 관계

DOM은 웹 문서의 구조와 상호 작용하는 기술 중 하나이다.
그러한 기술 중에 캐스케이딩 스타일 시트(Cascading Style Sheets), 간단히 CSS는 문서의 내용을 어떻게 표시할 것인가를 나타내는 데 사용하는 방법이다.

자바스크립트처럼 CSS는 문서의 <head>안에 <style>태그로 선언하거나 외부 스타일 시트 파일을 포함할 수 있다.
CSS를 표시하는 문법은 자바스크립트 함수와 매우 유사하다.

스타일 선언은 웹 브라우저에 요소를 표시하는데 사용하는 특정 색상이나 글꼴, 크기를 나타낼 수 있다.

상속(Inheritance)는 CSS의 가장 강력한 기능이다.
DOM처럼 CSS도 역시 노드 트리로 문서 내용을 볼 수 있다.
노드 트리 안에 있는 요소들은 그들 부모의 스타일 속성을 그대로 전달 받는다.

예를 들어 body 요소의 글꼴과 색상을 선언했다면 body가 포함하고 있는 모든 요소들에는 같은 글꼴과 색상이 자동 적용된다.

선언된 색상들은 <body> 태그가 포함하는 모든 내용에 적용할 수 있다.
(<body> 태그 안의 요소도 마찬가지이다.)

문서에 스타일을 적용할 때 특정 요소에만 적용하고 싶을 때가 있다.
다른 단락에는 영향을 주지 않고, 하나의 단락에만 다른 크기와 색상을 적용할 때 말이다.
이렇게 특정 단락만 영향을 주고 싶다면, 그 단락을 지정하고 알 수 있는 무언가를 집어넣어야 한다.
이때 사용하는 방법이 클래스(class)와 아이디(id)를 지정하는 것이다.

> 아이디 (id)

id 속성은 웹 페이지에서 유일하게 한 번 특정 요소를 지정해서 사용할 수 있다.

HTML에서 id 속성이 특정 요소에 단 한 번 선언댔다 하더라도 그 특정요소에 포함된 다른 요소들에 대한 스타일도 함께 지정할 수 있다.

DOM과 CSS가 같은 지시자(hook)를 사용하기 때문에 id 속성은 CSS가 지정한 지시자와 같은 방식으로 동작한다.

getElementById

getElementById라는 DOM의 메서드는 특정 아이디를 가지고있는 요소 노드에 직접 접근할 수 있다.
getElementById를 사용하면 특정 아이디를 사용하는 요소의 객체를 얻어낼 수 있다.

이 메서드는 문서 객체와 함께 사용할 수 있다.
getElementById는 하나의 파라메터 id를 받는다.

이렇게 하면 HTML 문서에서 purchases라고 지정된 아이디 속성을 포함하는 단 하나의 HTML 요소를 참조하게 된다.
이 요소는 하나의 객체가 되는 것이다.
그 결과를 typeof 연산자를 통해 이것이 객체인지, 문자열인지, 함수인지 확인할 수 있다.

문서 내에 모든 요소는 객체이다.
DOM을 사용해 이런 요소들 중 필요한 하나를 얻어낼 수 있다.

getElementsByTagName

getElementsByTagName을 사용하면 특정 태그를 사용하는 요소들을 배열로 얻어낼 수 있다.
(문서 내에 특정 태그들을 순서대로 저장한 묶음을 가져온다.)
getElementsByTagName도 하나의 파라메터 받는다.

반환된 배열 안 원소의 모든 값은 객체이다.

만약 특정 태그가 한 개만 있더라도 getElementsByTagName은 배열을 반환한다.

속성 다루기

문서 내에 모든 요소는 객체라는 사실이 중요하다.

  • 문서는 트리 구조를 지닌 객체들의 모음이다.
  • 요소, 속성, 텍스트 등 다양한 형식의 노드가 있다.
  • getElementById를 사용하면 특정 아이디를 가진 요소에 바로 접근할 수 있다.
  • getElementsByTagName을 사용하면 특정 태그를 가진 요소에 바로 접근할 수 있다.
  • 이러한 노드들은 모두 객체다.


- getAttribute

요소 노드에 접근하기 위해서는 getElementById나 getElementsByTagName이라는 두 가지 방법을 사용했다.
원하는 요소에 접근하게 되면 각 속성 값들을 알 수 있어야한다.
이때 사용하는 것이 getAttribute다.

getAttribute도 메서드이다.
또한 속성명을 파라메터로 갖는다.

위 두가지 메서드와는 달리 getAttribute는 문서 객체(document)를 사용할 수 없다.
각 요소 노드 객체로만 사용이 가능하다.

만약 title 속성이 없다면 getAttribute는 null을 반환한다.

- setAttribute

setAttribute는 특정 속성 노드의 값을 바꿀 수 있다.
getAttribute처럼 이 메서드도 함수이고 요소 노드에서만 동작한다.
다른 점은 파라메터를 두 개 갖는다.
하나는 바꾸고자 하는 속성이고, 다른 하나는 바꿀 값이다.

이것은 DOM을 통해 이미 표시된 웹 페이지의 내용을 동적으로 바꾸는 것이기 때문이다.
즉 DOM의 강력한 힘은 웹 브라우저가 문서를 새로고침 하지 않고도 내용을 바꿀 수 있다는 것이다.
