class 와 abstract 그리고 interface 이야기

2018-01-19 17:01:52
Programming>Story

왜 Class를 사용할까?

어떤 자동차 회사에서 똑똑하지만 무척 게으른 개빌자를 고용하였다.

한 자동차가 생산 될때마다 개발자는 아래와 같이 코드를 넣어주었다.

(이 세상 코드가 아니다.)

그런데 매번 자동차가 생산 될때마다 이 코드를 쳐야되는게 너무 귀찮았다.

개발자는 class라는 것을 만들었다.

개발자는 매번 자동차가 생산 할 때마다 모두 치지 않고, class를 이용하였다.


왜 abstract 상속을 사용할까?

이번에 회사에서 X가 아닌 B모델의 자동차와 C모델의 자동차를 둘다 생산한다고 했다.
그런데 B모델에는 부스터가 있고, C모델에는 브레이크가 있다.
개발자는 class를 만들다가, 나중에 다른 모델까지 생기는 것을 생각하였다.

개발자는 abstract를 만들었다.

그리고 이것을 extends하여 각 모델의 자동차 class에 넣어주었다.

이제 앞으로 어떤 모델이든 쉽게 class를 만들 수 있게 되었다.


왜 interface 를 쓸까?

자동차 회사에서 일 잘한다는 것을 듣고 옆 배만드는 회사와 비행기, 기차회사에서 일을 해달라고 요청이 들어왔다.
개발자는 class와 abstract를 넘기려고 했더니 해당 회사들에서 사용 할 수 없는 것들이 있었다.
예를들어 달린다{}; 의 경우 어디서 달리는지는 다르지만 의미는 같은 것이었다.
잠시 생각한 개발자는 더 큰 틀로 abstrct를 바라보고 한번 더 나누어 보았다.

개발자는 interface라는 것을 만들었다.

그리고 이것을 implement를 이용하여 각 abstract에 적용하였다.

이로서 아주 쉽게 변경하여 각 회사에 넘겨 줄 수 있었다.


결론

결과적으로 class, abstract, interface 모두 개발자의 편리성 때문에 생겨난 것이다.
그리고 재사용성 뿐만아니라 더 많은 일을 할수 있게 되었다.
