자바스크립트는 type 이 없다?

2018-01-08 23:46:56
Dev>Javascript


자바스크립트 같은 동적언어에는 type 이 없다.


자바스크립트 개발자가 보면 ‘읭?’ 할 만한 문구이다.
이 문구를 보고 처음에 ‘읭?’ 했다가, ‘아 그렇게 느낄수도 있겠구나’ 라고 생각했다.

다른 언어에는 변수선언시 int float boolean 등으로 선언을 하나,
자바스크립트는 대부분 var 로 모두 선언을 하기 때문이다.

그리고 대부분 이러한 코드에서 그렇게 느낄 수도 있을거라 생각한다.

하지만 위 현상은 자바스크립트에서 일어나는 특징이지 같은 타입은 아니다.

자바스크립트에는 다음과 같은 타입이 있다.

  • null
  • undefined
  • boolean
  • number
  • string
  • object
  • symbol (ES6 부터 추가)


‘boolean, string, number 는 알겠는데, 그 이외에 저것들은 뭐야? 심지어 null 도 타입이야?’


그렇다.
그러면 변수가 무슨 타입인지 확인 해 볼수없을까?

자바스크립트에는 typeof 라는 연산자가 있다.


‘null 도 타입이라면서 null은 왜 object야?’


이것은 일종의 ‘버그’ 라고 칭해진다.

물론 ‘null’을 반환하도록 수정요청이 있었지만 초기에 자바스크립트로 작성한 프로그램에 문제가 있을수 있기 때문에 수정하지 않았다고 한다.

‘그러면 typeof 를 사용해 해당 타입이 null 인경우만 체크하고 싶은데 방법은 없는가?’


그건 null의 특징을 이용하면 체크 할 수 있다!

null은 false 값이면서 타입이 object 이다.


결론을 말하면 자바스크립트는 엄연히 타입이 존재한다.
