Javascript>> null vs undefined

2018-01-09 09:52:06
Javascript>Basic


자바스크립트에는 특별한 값이 있다.

바로 undefined 라는 값이다.

undefined 는 값이없는 것을 의미한다.


그런데 비슷한 것으로 null 이 있지 않은가? 무엇이 다른가?

null과 undefined는 둘다 값이 없는것을 의미하는것이 맞다.
그러나 null은 ‘null’ 이라는 값을 할당 한것이다.

앞에서 null은 typeof로 확인시 ‘object’ 라고 나오는것을 기억하는가?

아마도 메모리상에 null 은 static상태로 저장된 상태라 생각한다.
그래서 할당시 실제로 있는 값이기 때문에 ‘object’ 라 나오는것같다.



비교를 하면 어떨까?


== 연산자에선 두 값이 같다고 나온다.


나중에 포스팅 하겠지만 좀 더 엄격하게 비교도 가능하다.


“그런데 선언되지 않은 변수는 그럼 어떤값이지?”


브라우저 콘솔상에선 선언되지 않은 변수에 대해선 이렇게 나온다.

‘b is not defined? 이건 완전 undefined 란 의미 아닌가?’
언어적으로는 맞는 말이긴하나 자바스크립트 내에선 다른 의미이다.
처음부터 그냥 ‘b is not found’ 라고 하지…


게다가 typeof를 사용하면 더 헷갈린다!
