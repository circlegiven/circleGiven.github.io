컴퓨터 프로그램의 구조와 해석

2018-01-12 10:12:15
Programming>SICP

‘컴퓨터 프로그램의 구조와 해석 (SICP)’을 읽으며 정리한 글이다.
이 책의 컨셉은 ‘마법사’다.
그리고 책에서는 Scheme(Lisp의 사투리)라는 프로그래밍 언어를 쓴다.



프로세스 Process

사람이 만든 규칙(프로그램)에 따라 움직 이는 것, 움직이는 과정

마법사가 마법을 일으키는데 주문을 말하는 과정



계산 프로세스 Computational Process

데이터라고 하는것을 조작하면서 어떤 일(프로세스)을 한다.



프로그램 Program

프로세스가 일어나도록 사람이 만든 규칙

주문



프로그래밍 언어 Programming language

프로세스가 작동하도록 하는 프로그램은 여러 식으로 이루어지는데, 이런 식을 적을때 사용하는 말이 프로그래밍 언어이다.

주문을 만드는 언어



전체적인 과정

프로그래밍 언어로 작성된 프로그램 >> 프로세스 >> 결과

특정언어로 만든 주문 >> 주문 말하기 >> 마법



오류잡기 Debug

프로그램속에서 문제를 일으키는 오류가 어디에 숨어있는지 찾아내고 고치는 과정

주문이 제대로 일어나지 않는 다면 주문이 어디가 잘못됬는지 알아야 된다



프로그램 짤 때 바탕이 되는것

프로그래밍 언어는 프로세스에 대한 사람의 새악을 짜임새 있게 담아내는 그릇이다.
그러므로 언어를 설명할때에는 단순한 생각을 모아 복잡한 생각을 처리 해야된다.

마법을 쓰는데 주문이 욕나오게 어렵다고 생각하자. 그러면 쉬운 주문으로 하도록 해야된다.

좋은 프로그래밍언어는 아래와 같은 일을 할 수 있다.
기본식 - 언어에서 가장 단순한 것
엮어내는 수단 - 간단한 것(기본식)을 모아 복잡한 것으로 만듬
요약하는 수단 - 복잡한 것(엮어내는 수단)에 이름을 붙여 하나로 다룰수 있도록 간추림

그리고 프로그램을 짤때 두가지가 사용된다.

데이터 - 프로시저에서 쓰려는 물건
프로시저 - 데이터를 처리하는 규칙


식

여기서 +는 연산자 이다.
137과 349는 피연산자가 된다.
이 둘을 조합한 것을 엮은 식 이라 한다.

엮은 식 = 연산자 + 피연산자

그리고 이 엮은 식을 계산한 값 486은 프로시저에 인자를 넘겨 얻는다.

즉, 연산자는 프로시저가 되고 피연산자는 인자가 된다.

변수와 환경 Variable & Environment

여기서 define은 이름을 정의한다는 뜻이다.
size는 define에 의해 정의된 이름으로 변수라 불린다.
그리고 2는 변수의 값이 된다.


그런데 size를 불러오면 2라는 값이 나온다는것은, 어딘가에 저장해둔 메모리가 있다는 뜻이다.
이런 기억공간을 환경 이라고 한다.


프로시저 Procedure

앞서 프로시저는 데이터를 처리하는 규칙이라 하였다.
이번엔 프로시저를 정의해 보겠다.

이건 square라고 불리우는 프로시저를 정의한 형태이다.
x를 인자로 받아들여 값을 나타낼 것이다.

이미 있던 다른 프로시저들을 하나로 묶어 만든 프로시저를 묶음 프로시저(compound procedure)라고 한다.
위에서 다른 프로시저는 바로 *연산자가 된다.

그러므로 square는 묶음 프로시저가 된다.


조건 식과 술어 Predicate

만약 조건이 들어가는 경우는 어떻게 표현해야될까?
절대값과 같은 경우 말이다.

여기서 cond 옆으로 나열된 것을 절(clause)라고 한다.
그리고 (> x 0)처럼 결과가 참 또는 거짓을 나타내는 식을 술어(predicate)라고 한다.
그 옆에 x는 술어의 짝인 결과 식(consequent expression) 이라고 한다.

cond와 비슷하게 이런식으로도 가능하다.

여기서 (< x 0)은 술어가 되고 (- x)와 x는 결과 식 이 된다.
